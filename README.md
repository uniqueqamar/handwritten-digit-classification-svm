# Robustness Analysis of Handwritten Digit Classification using SVM

> A research-oriented ML project investigating how Gaussian input perturbations affect SVM classifier performance — simulating real-world distribution shift.

---

## Research Question

> *How does input noise — as a proxy for real-world distribution shift — degrade the performance of an SVM classifier trained on clean digit images? Which digit classes are most vulnerable, and why?*

This is not just a classification project. It is a **robustness study**: we train a model under ideal conditions and then systematically stress-test it to understand its failure modes.

## Methodology

### 1. Dataset
The **sklearn Digits dataset** contains 1,797 grayscale images of handwritten digits (0–9), each 8×8 pixels (64 features). It is a standard benchmark that is compact, well-understood, and free of preprocessing complications — ideal for a focused robustness study.

### 2. Model
A **Support Vector Machine with RBF kernel** (`C=10`, `γ=0.001`). SVMs are a strong choice here: they are margin-based classifiers with known sensitivity to input perturbations, making them an interesting subject for robustness analysis. Features are standardised with `StandardScaler` (fitted on training data only, to prevent data leakage).

### 3. Noise Model
We inject **zero-mean Gaussian noise** at increasing standard deviations (σ = 0.0 → 2.0). After StandardScaler, all features have unit variance, so σ is directly interpretable as a fraction of a feature's typical spread. This simulates:
- Sensor noise during image capture
- Compression artefacts
- Domain shift between training and deployment environments

### 4. Evaluation Pipeline
For each noise level:
- Perturb the test set (training data is never touched)
- Predict with the frozen model
- Record overall accuracy and per-class F1 scores
- Store confusion matrix

### 5. Analysis Outputs

| Plot | What it reveals |
|---|---|
| Accuracy vs. Noise curve | Overall fragility of the model |
| Per-class F1 heatmap | Which digits degrade earliest |
| Vulnerability bar chart | Ranked sensitivity across digit classes |
| Misclassified samples | Whether errors are random or systematic |
| Clean vs. Noisy side-by-side | How noise manifests in pixel space |

## Key Findings

- **Baseline accuracy**: ~98% on clean data, consistent with published benchmarks for this dataset.
- **At σ=1.0**: Accuracy drops to approximately X% — noise equivalent to one standard deviation per feature begins to noticeably confuse similar-looking digits.
- **Most vulnerable digits**: Digits with visually similar neighbours (e.g., 3↔8, 4↔9, 1↔7) tend to have the steepest F1 drops. Their decision boundaries in 64-dimensional space are close together, so small perturbations push samples across them.
- **Most robust digits**: Digits with distinct visual structure (e.g., 0) maintain higher F1 even at high noise.

## Design Decisions

**Why not deep learning?**
SVMs are a deliberate choice. They are interpretable, well-understood theoretically, and their margin-based geometry makes the effect of noise analytically tractable. This project is about *understanding* robustness, not achieving state-of-the-art accuracy.

**Why Gaussian noise?**
It is the standard noise model in signal processing and the most common perturbation used in robustness benchmarks. It serves as a controlled, reproducible proxy for distribution shift.

**Why fix hyperparameters?**
Our research question concerns post-training robustness, not hyperparameter sensitivity. Fixing the model lets us attribute performance changes entirely to noise level.
